# Contributor License and Rights Statement

ACGS-Lite is distributed under the Apache-2.0 license. Contributions submitted to this repository are expected to be compatible with that license.

This repository currently uses a lightweight contribution-rights certification in the pull request template. A stricter Individual CLA / Corporate CLA workflow can be added later without changing the development policy described here.

## Individual contributors

By opening a pull request, an individual contributor certifies that:

1. They created the contribution or otherwise have the right to submit it.
2. The contribution may be distributed under the repository license.
3. The contribution does not knowingly include restricted third-party code or material without compatible permission.
4. The contributor understands that the public GitHub record is retained as review and provenance evidence.

## Corporate contributors

If the contribution is made as part of employment, consulting, client work, sponsorship, or another organization-directed activity, the submitter certifies that:

1. The organization has authorized the contribution or the contributor has sufficient authority to submit it.
2. The contribution may be distributed under the repository license.
3. Any organization-specific attribution, notice, or compatibility requirement is included in the pull request.

## Maintainer handling

Maintainers should block merge when the contribution-rights section of the pull request is incomplete, disputed, or inconsistent with the changed files. For material licensing questions, open a maintainer discussion before merge.

## Future automation

A CLA bot may later enforce signed Individual CLA and Corporate CLA records. Until then, the pull request template and policy checks are the minimum required evidence floor.
