# LinkedIn Post: Microsoft validated the category. ACGS is the constitutional layer.

Microsoft’s Agent Governance Toolkit launch is good news for everyone building in AI governance.

Why?

Because it validates something important:

**agent governance is now a real category.**

But it also sharpens a distinction I think matters.

A lot of governance tooling is really about **policy enforcement**:
- intercept an action
- evaluate it against rules
- allow or deny

ACGS is built around a stricter question:

**was the agent constitutionally permitted to take that action in the first place?**

That leads to a different architecture:
- fail-closed by default
- MACI separation of powers
- constitutional hashing
- amendment lineage
- governed workflows like block, audit, escalate, and human review
- regulator-facing audit evidence

That distinction matters more as agents move from chat to:
- approvals
- deployments
- procurement actions
- compliance workflows
- operational decisions

Once agents can act, governance stops being only about output safety.
It becomes about **authority, oversight, and proof**.

That is where ACGS is trying to go deeper.

**Microsoft validated runtime governance as a market.**
**ACGS is aiming at constitutional governance inside that market.**

If that framing resonates, I’d love to compare notes with people thinking about:
- AI oversight architectures
- fail-closed governance
- EU AI Act evidence requirements
- separation of duties for agent systems
