## EVAL: hackathon-demo-project

### Context
The demo is the hackathon video: a Duo agent generates code, opens an MR, the Constitutional
Sentinel detects violations, posts inline comments, and blocks the merge.
These evals validate the demo scenario works correctly before recording.

---

### Demo Script
1. Duo agent generates Python code with hardcoded secret (`password = "abc123"`)
2. Agent opens MR titled "AI: Add user authentication"
3. MR hook triggers → Cloud Run webhook → governance pipeline
4. Report: FAILED, risk_score > 0.7, 1+ CRITICAL violation
5. Inline comment on the diff line with the secret
6. "Merge Blocked" comment posted
7. MACI check: author ≠ approver enforced

#### CAP-DEMO-01: AI-generated code with secret triggers CRITICAL violation
```bash
python -c "
from acgs_lite.engine import GovernanceEngine
from acgs_lite.constitution import Constitution

engine = GovernanceEngine(Constitution.default())

# Simulated AI-generated code (what Duo might write)
ai_code = '''
def authenticate(username, password):
    DB_PASSWORD = \"abc123secret\"  # hardcoded credential
    if password == DB_PASSWORD:
        return True
    return False

critical = [v for v in result.violations if v.severity.value == 'critical']
assert len(critical) > 0, f'No CRITICAL violations found. Got: {[v.rule_id for v in result.violations]}'
print(f'PASS — {len(critical)} CRITICAL violation(s): {[v.rule_id for v in critical]}')
" && echo "PASS" || echo "FAIL"
```

#### CAP-DEMO-04: Diff parser correctly identifies the secret line
```bash
python -c "
from acgs_lite.integrations.gitlab import _parse_added_lines

# Simulated GitLab MR diff for AI-generated auth.py
demo_diff = '''@@ -0,0 +1,6 @@
+def authenticate(username, password):
+    DB_PASSWORD = \"abc123secret\"
+    if password == DB_PASSWORD:
+        return True
+    return False
+
'''

lines = _parse_added_lines(demo_diff)
secret_lines = [(n, l) for n, l in lines if 'abc123secret' in l]
assert len(secret_lines) == 1, f'Expected 1 secret line, got {secret_lines}'
line_no, content = secret_lines[0]
assert line_no == 2, f'Expected line 2, got {line_no}'
print(f'PASS — secret detected at line {line_no}: {content.strip()}')
" && echo "PASS" || echo "FAIL"
```

#### CAP-GREEN-02: Batch validation more efficient than per-line
```bash
python -c "
import time
from acgs_lite.engine import GovernanceEngine
from acgs_lite.constitution import Constitution

engine = GovernanceEngine(Constitution.default())
lines = ['line of code ' * 5] * 100

# Per-line validation
t0 = time.perf_counter()
for line in lines:
    engine.validate(line, agent_id='bench')
per_line_ms = (time.perf_counter() - t0) * 1000

# Batch (single join)
t1 = time.perf_counter()
engine.validate('\n'.join(lines), agent_id='bench-batch')
batch_ms = (time.perf_counter() - t1) * 1000

print(f'Per-line: {per_line_ms:.1f}ms | Batch: {batch_ms:.1f}ms')
# Batch should be faster (fewer overhead calls)
# Not strictly enforced — just diagnostic
print('PASS')
" && echo "PASS" || echo "FAIL"
```
