# Constitutional Sentinel — Demo Video Script

**Target: 3 minutes** (hard cap per DevPost rules)

---

## SECTION 1: The Problem (0:00 – 0:30)

### Show
- Slide or terminal: "AI Agents Are Writing Your Code. Who's Watching?"
- Quick `git log --oneline` showing AI-generated commits

### Say
> "AI coding assistants generate over 40% of new code at many organizations.
> They're fast, productive — and they have no idea what your compliance rules are.
>
> An AI agent can hardcode an API secret, leak a Social Security number, or drop a
> production table. If the only reviewer is the developer who prompted the AI, nobody catches it."

### Say

> But these comments weren't left by a human. The Constitutional Sentinel — an independent

>
> Hardcoded AWS key on line 14. Social Security number on line 25. DROP TABLE on line 36.
> Twelve violations total. Every single one caught."

## SECTION 3: Governance Report (1:15 – 1:50)

### Show
1. Scroll to MR discussion — governance report comment
2. Zoom to 150% on: Risk Score 0.95, constitutional hash, violation table
3. Highlight "Merge Blocked" comment

### Say
> "The Sentinel posted a governance report. Risk score: 0.95 out of 1.0.
>

>

---

## SECTION 4: Architecture (1:50 – 2:30)

### Show
1. Architecture diagram: GitLab → Cloud Run → ACGS-Lite Engine → Comments back to GitLab
2. Three roles: Proposer (Duo) → Validator (Sentinel) → Executor (Human)
3. Terminal: `curl -s https://acgs-sentinel-208702602468.us-central1.run.app/health | python3 -m json.tool`

### Say
> "When a merge request opens, a webhook fires to the Constitutional Sentinel on Google Cloud Run.
> It pulls the diff, validates every line through ACGS-Lite at 560 nanoseconds per line, and posts
> violations back as inline comments.
>
> The key innovation: MACI — separation of powers for AI. The Proposer writes code. The Validator
> reviews independently. The Executor — a human — decides to merge. No agent validates its own output.
>
> The constitution is YAML — any organization can define their own rules: HIPAA, SOC 2, internal
> security policies. Instant, tamper-evident enforcement."
