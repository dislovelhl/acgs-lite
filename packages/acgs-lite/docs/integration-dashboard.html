Gap Analysis — Prioritized

Critical Gaps (P0)

No Visual Dashboard / UI
No web UI for real-time governance monitoring, audit trail browsing, or compliance scoring. The propriety-ai SvelteKit app has no governance dashboard components. Everything is CLI or programmatic.
~3-4 weeks SvelteKit or standalone React dashboard
No OpenTelemetry / Prometheus Export
Governance metrics (validation count, violation rate, latency p50/p99, compliance score) cannot be scraped or traced. Cloud Logging exists but no standard observability stack integration.
~1 week OTel SDK + Prometheus exporter
No Streaming Response Governance
All integrations validate complete responses. No incremental token-by-token or chunk-level governance for streaming LLM outputs. High risk for real-time chat applications.
~2 weeks Sliding window validator for SSE/WebSocket streams
No GitHub Actions Integration
GitLab CI/CD is fully supported with webhook handler, MACI enforcement, and .gitlab-ci.yml generator. GitHub Actions — the larger market — has zero support.
~1 week Action YAML + webhook handler
Compliance Frameworks Are Checklists, Not Automation
All 9 frameworks define checklist items with hand-coded statuses. No automated evidence collection from running systems. Gap between "we track this" and "we can prove it in an audit."
~4-6 weeks Evidence collector + framework-specific probes

High Gaps (P1)

Missing Framework Adapters: CrewAI, Semantic Kernel, DSPy, Pydantic AI, Haystack
Five popular AI frameworks have no governance wrapper. CrewAI and Pydantic AI are particularly high-growth. Pattern is well-established — each adapter is ~100-200 LOC.
~2-3 days per adapter, parallelizable
No Multi-Turn Conversation State Tracking
Governance validates individual messages, not conversation trajectories. A compliant message might become a violation in context of prior messages (e.g., gradual prompt injection).
~2 weeks Conversation context window + trajectory scoring
No Notification / Alerting Channel
Violations are logged but never push-notified. No Slack, Teams, email, PagerDuty integration for governance alerts. Compliance teams need real-time violation alerts.
~3-5 days Webhook + Slack/Teams adapter
Constitution Advanced Modules: 80 files, ~20% avg implementation
Lifecycle (amendments, waivers, change requests), enforcement (circuit breaker, quarantine), monitoring (drift, anomaly) exist as framework specs but aren't production-usable.
~6-8 weeks Prioritize: amendments, emergency override, drift detection

Medium Gaps (P2)

No Formal Schema Definitions
schema/ directory exists but is empty. Schema validation happens inline. No JSON Schema or OpenAPI spec for governance payloads.
~2-3 days
No Multi-Tenant Isolation Testing
Constitution and engine are per-instance, but no explicit tenant isolation guarantees. No cross-tenant data leakage tests for SaaS deployment.
~1 week
Regional Compliance: UK AI Bill, Singapore PDPA
EU AI Act has 40% coverage. Other jurisdictions at 0%. Growing market demand as AI regulation spreads globally.
~1 week per jurisdiction
Load Testing (Sustained Throughput)
Microbenchmarks show ~560ns p50 but no sustained load testing at 1M+ req/hr. No memory profiling under pressure.
~3 days locust/k6 harness

Low / Strategic Gaps (P3)

ML-Based Rule Synthesis
Rules are manually authored. No automated rule suggestion from audit trail patterns or violation clusters.
~4 weeks Research + prototype
Formal Verification of Rule Consistency
Conflict detection exists (basic) but no formal SMT-based verification that rules don't contradict each other.
~3 weeks Z3 integration
K8s Operator / Helm Chart
Cloud Run deployment works. No K8s-native deployment or operator for governance sidecar injection.
~2 weeks
Interactive Constitution Editor
Constitutions are YAML files. No visual rule editor, conflict preview, or impact simulation UI.
~4-6 weeks
